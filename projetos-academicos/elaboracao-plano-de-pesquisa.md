# 📘 Modelo de Plano de Pesquisa  
Syrran Tech Academy  

Este documento orienta a elaboração do **Plano de Pesquisa** para projetos acadêmicos.  
O objetivo é organizar, planejar e estruturar corretamente o desenvolvimento do projeto.

---

# 🏷 Título do Projeto

> Escreva um título claro, direto e objetivo.

O título deve indicar:
- O que será desenvolvido
- Para quem ou para qual finalidade
- O contexto do projeto

Evite títulos genéricos como:
- "Sistema Web"
- "Aplicativo Escolar"

Prefira:
- "Desenvolvimento de Sistema Web para Gestão de Doações"
- "Aplicativo Mobile para Organização de Rotinas Escolares"

---

# 📂 Categoria

Marque a categoria que melhor representa o foco principal do projeto:

- (    ) Ciências da Natureza e Exatas  
- (    ) Informática  
- (    ) Ciências Humanas e Linguagens  
- (    ) Engenharias  

Projetos de software geralmente se enquadram em **Informática**.

---

# 📝 Resumo

O resumo deve conter entre **8 e 15 linhas**, em parágrafo único.

Ele deve responder:

- O que será desenvolvido?
- Por que o projeto é importante?
- Com base em que dados ou estudos?
- Quais resultados foram ou serão obtidos?

O texto deve ser:
- Claro
- Objetivo
- Sem citações longas
- Sem detalhamento excessivo

---

# 🔑 Palavras-Chave

Indicar de **3 a 5 palavras-chave** que representem o tema central do projeto.

Exemplos:
- Sistema Web
- Aplicação Mobile
- Banco de Dados
- Desenvolvimento de Software
- Responsividade

Evite termos genéricos como:
- Projeto
- Tecnologia

---

# 📌 Plano de Pesquisa

---

## 📖 Introdução

A introdução deve:

- Apresentar o tema
- Explicar o problema
- Justificar a importância do projeto

Responder:

- Qual problema será resolvido?
- Quem é afetado?
- Por que a solução é necessária?

Recomenda-se entre **1 e 2 páginas**.

---

## 🎯 Objetivos

### Objetivo Geral

Deve expressar o propósito principal do projeto.

Começar com verbo no infinitivo:

- Desenvolver
- Criar
- Implementar
- Construir
- Propor
- Analisar

Exemplo:
> Desenvolver um sistema web para organizar o cadastro de animais resgatados.

---

### Objetivos Específicos

São etapas menores que conduzem ao objetivo geral.

Exemplo:

- Levantar requisitos do sistema
- Modelar o banco de dados
- Desenvolver a interface
- Implementar funcionalidades principais
- Realizar testes

Sugere-se entre **3 e 6 objetivos específicos**.

---

## 🛠 Metodologia

Descrever como o projeto será desenvolvido.

Indicar:

- Tipo de pesquisa (exploratória, aplicada, bibliográfica)
- Ferramentas utilizadas
- Etapas do desenvolvimento
- Forma de coleta de dados (se houver)

A metodologia deve demonstrar organização e viabilidade.

---

## 📅 Cronograma

Organizar as etapas no tempo.

Pode ser dividido por:

- Meses
- Bimestres
- Semanas

Exemplo:

- Mês 1 – Pesquisa e levantamento de requisitos  
- Mês 2 – Desenvolvimento inicial  
- Mês 3 – Testes e ajustes  
- Mês 4 – Finalização e relatório  

O cronograma deve ser realista.

---

## 📊 Resultados Esperados

Descrever:

- O que será entregue
- Qual impacto o projeto terá
- Quem será beneficiado
- Que problema será resolvido

Exemplo:

- Sistema funcional
- Interface responsiva
- Organização eficiente de dados
- Redução de erros manuais

---

## 📚 Referências

Listar livros, artigos, sites e autores utilizados.

Utilizar fontes confiáveis.

Evitar:
- Blogs sem credibilidade
- Wikipédia como única fonte

Sempre que possível, utilizar:
- Livros técnicos
- Artigos científicos
- Documentação oficial das tecnologias utilizadas

---

# ✅ Observações Finais

O Plano de Pesquisa deve demonstrar:

- Clareza
- Organização
- Coerência
- Viabilidade

Este documento é a base para o desenvolvimento do produto e do relatório final.
